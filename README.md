# .github – Estándares de Desarrollo

Este repositorio centraliza los **estándares, workflow y lineamientos**
utilizados por el equipo para todos los repositorios de la organización.

Los documentos acá contenidos son el **Estandar** sobre cómo se trabaja y cómo se integran los cambios hasta producción.

## Contenido
- **WORKFLOW.md** – Workflow oficial (Issues, ramas, integración, TST/PROD, Milestones, Tags)
- **ONBOARDING.md** – Guía rápida para desarrolladores
- **LEGACY_VS_NEW.md** – Reglas y cuidados para repositorios legacy
- **PULL_REQUEST_TEMPLATE.md** – Template liviano de Pull Requests

## Nota sobre Issue Templates

Los Issue Forms (`.yml`) se encuentran bajo la ruta:

.github/ISSUE_TEMPLATE/

dentro de este repositorio.

Esta estructura es **intencional y necesaria** para que GitHub aplique los templates a todos los repositorios de la organización.
