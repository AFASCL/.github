# .github – Estándares de Desarrollo

Este repositorio centraliza los **estándares, workflow y templates**
compartidos por todos los repositorios de la organización.

Estos documentos son el **esquema oficial** sobre cómo trabajamos.

## Contenido
- **WORKFLOW.md** – Esquema oficial de trabajo (Issues, ramas, PRs, TST/PROD, Milestones, Tags)
- **ONBOARDING.md** – Guía rápida para desarrolladores nuevos
- **LEGACY_VS_NEW.md** – Reglas y cuidados para repositorios legacy vs nuevos
- **ISSUE_TEMPLATE/** – Issue Forms oficiales (Bug / Feature / Chore)
- **PULL_REQUEST_TEMPLATE.md** – Template liviano de Pull Requests

## Objetivo
- Tener reglas claras y únicas
- Evitar confusión entre repos
- Facilitar onboarding
- Mantener trazabilidad completa hasta PROD
